## 1. KEYWORD STRATEGY

### 1.1 Strategic Overview

Carbonium operates in a high-intent, commercially competitive niche: AI-driven marketing services in Dubai and the UAE. The keyword strategy targets three layers:

1. **Brand + location** — own "Carbonium" + "Dubai/UAE" searches from day one
2. **Service + location** — capture high-intent buyers searching for specific AI marketing services in the region
3. **Informational + educational** — blog content that ranks for "how to" and "what is" queries and funnels readers to service pages

The GCC market has lower organic competition in English than Western markets but high commercial intent. Prioritise UAE/Dubai geo-modified keywords over generic global terms — they convert better and compete more realistically for a new domain.

## 2. META TAGS — ALL 6 PAGES

### Conventions

- **Title tag:** 50–60 characters, primary keyword near front, brand name at end
- **Meta description:** 150–160 characters, includes primary keyword, a benefit statement, and a CTA
- **Canonical:** Always self-referencing; adjust domain once confirmed
- **Open Graph:** Required for social sharing previews
- **Twitter Card:** Summary large image for all pages
- **Robots:** `index, follow` for all main pages; `noindex` for thank-you, privacy, 404

### 4.4 Core Web Vitals & Performance

Target: Lighthouse Performance ≥ 85 mobile (per project success criteria)

- [ ] **LCP (Largest Contentful Paint) < 2.5s** — Hero image lazy-loaded with `loading="eager"` for above-fold; `loading="lazy"` for all below-fold images
- [ ] **CLS (Cumulative Layout Shift) < 0.1** — All images and embeds have explicit `width` and `height` attributes; no font swap causing reflow
- [ ] **FID / INP < 200ms** — Avoid large JS bundles blocking the main thread; defer non-critical scripts
- [ ] **Images optimised** — All images served in WebP format with JPEG fallback; max 150KB for hero, 80KB for thumbnails
- [ ] **Fonts** — Self-host chosen font (Space Grotesk / Syne / Inter); use `font-display: swap`; preload critical font files with `<link rel="preload">`
- [ ] **CSS** — Critical CSS inlined in `<head>`; non-critical CSS deferred
- [ ] **JavaScript** — Animations using GPU-accelerated CSS transforms (`transform`, `opacity`) only; no layout-triggering JS in scroll handlers; `will-change` used sparingly
- [ ] **Reduced Motion** — All scroll animations wrapped in `@media (prefers-reduced-motion: reduce)` CSS query; animations disabled or minimal for users who opt out
- [ ] **Hosting** — Confirm CDN is active (Vercel/Netlify both include global CDN); enable Brotli/gzip compression
- [ ] **Cache headers** — Static assets (images, fonts, CSS, JS) cached with `Cache-Control: max-age=31536000, immutable`
